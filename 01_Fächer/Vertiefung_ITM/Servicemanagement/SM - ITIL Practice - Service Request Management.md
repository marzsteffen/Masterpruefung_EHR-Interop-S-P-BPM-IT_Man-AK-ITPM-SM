# SM - ITIL Practice - Service Request Management

> [!info] Kurzdefinition
> ITIL-4-Practice (Service Management) zum Unterstützen der vereinbarten Servicequalität durch effektive und benutzerfreundliche Bearbeitung aller vordefinierten, vom Anwender initiierten Service Requests. Ein Service Request ist eine Anfrage eines Anwenders, die eine als normaler Bestandteil der Servicebereitstellung vereinbarte Serviceaktion einleitet.

## Beschreibung

Folien 103–105 vertiefen die Practice. Quelle: ITIL 4 Foundation Courseware (Axelos / Van Haren Publishing 2019).

**Zweck (Folie 103):**
> „Das Unterstützen der vereinbarten Servicequalität, indem alle vordefinierten, vom Anwender initiierten Service Requests effektiv und benutzerfreundlich bearbeitet werden."

**Definition Service Request:**
> „Ist eine Anfrage eines Anwenders oder des Bevollmächtigten eines Anwenders, der eine Serviceaktion einleitet, die als normaler Bestandteil der Servicebereitstellung vereinbart wurde."

**Service Request Genehmigung (Folie 104):**
- Service Requests erfordern oft eine Autorisierung (finanzielle, datenschutzrechtliche oder andere Richtlinien), während andere keinerlei Autorisierung benötigen.
- Service Requests und ihr Fulfillment sollten standardisiert und soweit wie möglich automatisiert werden.
- Richtlinien sollten unabhängig davon festgelegt werden, welche Service Requests mit eingeschränkten oder sogar ohne zusätzliche Genehmigungen erfüllt werden, sodass ihre Erfüllung optimiert werden kann.
- Die Erwartungen von Anwendern im Hinblick auf Fulfillment-Zeiten sollten klar festgelegt werden und darauf basieren, was die Organisation realistischerweise liefern kann.
- Verbesserungsmöglichkeiten sollten identifiziert und implementiert werden, um kürzere Fulfillment-Zeiten zu realisieren und Automatisierung zu nutzen.
- Es sollte Richtlinien und Workflows für die Dokumentation und Weiterleitung aller Requests geben.

**Beispiele für Service Requests (Folie 105):**

- **Request für eine Servicebereitstellungsaktion** — z. B. Bereitstellen eines Berichts oder Ersetzen einer Tonerkartusche.
- **Eine Informationsantrage** — z. B. Wie erstelle ich ein Dokument? Oder: Wie sind die Bürozeiten?
- **Request zur Bereitstellung einer Ressource oder Service** — z. B. Bereitstellen eines Telefons oder Laptops für einen Anwender oder Bereitstellen eines virtuellen Servers für ein Entwicklungsteam.
- **Request zum Gewähren von Zugriff auf eine Ressource oder einen Service** — z. B. Bereitstellung von Zugriff auf eine Datei oder einen Ordner.
- **Feedback, Lob und Beschwerden** — z. B. Beschwerden über eine neue Benutzeroberfläche oder Lob für ein Support-Team.

## Bestandteile / Aufbau

| Aspekt | Inhalt |
|---|---|
| Zweck | Vereinbarte Servicequalität unterstützen, Standard-Requests effektiv/benutzerfreundlich bearbeiten |
| Definition Service Request | Anfrage des Anwenders, vereinbarte Serviceaktion |
| Genehmigung | Manche Requests benötigen Autorisierung (finanziell, Datenschutz, andere); manche nicht |
| Standardisierung | Soweit möglich automatisieren |

## Beispiel

Krankenhaus-Beispiele:
- Servicebereitstellung: neue Tonerkartusche im OP-Bereich
- Information: „Wie generiere ich einen Entlassbrief?"
- Ressource: Pflegekraft erhält Tablet
- Zugriff: neue Pflegekraft erhält KIS-Berechtigungen
- Feedback: Lob für neue mobile App; Beschwerde über UI-Änderung

## Abgrenzung

- **Service Request vs. Incident** ([[SM - ITIL Practice - Incident Management]]): Service Request = vereinbarter Standard-Service; Incident = ungeplante Unterbrechung oder Qualitätsminderung.
- **Service Request Management vs. ITIL V3 Request Fulfillment** (siehe [[SM - ITIL V3 Practice - Request Fulfillment]]): Inhaltlich sehr ähnlich, in ITIL 4 explizit als Practice mit Outcome-Fokus.
- **Service Request vs. Change Request**: Service Request = Standard, vereinbart; Change Request = Änderung der IT-Landschaft (Change Enablement).

## Prüfungsrelevanz

**Typische Definitionsfrage:** Was ist der Zweck der Service Request Management Practice? — Vereinbarte Servicequalität durch effektive und benutzerfreundliche Bearbeitung aller vordefinierten Service Requests unterstützen.

**Definition Service Request:** Anfrage eines Anwenders zur Einleitung einer als normaler Bestandteil der Servicebereitstellung vereinbarten Serviceaktion.

**Diskussionsfrage:** Welche Beispiele für Service Requests gibt es? — Servicebereitstellung, Information, Ressource, Zugriff, Feedback (Folie 105).

**Mögliche Anschlussfragen:**
- Wie unterscheidet sich Service Request von Incident?
- Welche Rolle spielt die Standardisierung — und welche Beziehung zum Servicekatalog?
- Wann braucht ein Service Request eine Autorisierung?
- Wie unterscheidet sich Service Request Management in ITIL 4 vom ITIL-V3-Pendant Request Fulfillment?
