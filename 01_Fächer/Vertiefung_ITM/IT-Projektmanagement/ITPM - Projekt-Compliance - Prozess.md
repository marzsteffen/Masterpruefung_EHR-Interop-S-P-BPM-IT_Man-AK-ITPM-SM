# ITPM - Projekt-Compliance - Prozess

> [!info] Kurzdefinition
> Der projektbezogene Compliance-Prozess umfasst sechs Schritte: (1) Identifizierung relevanter Regelwerke, (2) Ableitung von Compliance-Anforderungen, (3) projektbezogene Compliance-Organisation, (4) Information der Projektbeteiligten, (5) Dokumentation, (6) Reaktion auf neue Compliance-Entwicklungen.

## Bestandteile / Aufbau

```
1. Identifizierung relevanter Regelwerke
   ↓
2. Ableitung von Compliance-Anforderungen
   ↓
3. Projektbezogene Compliance-Organisation
   ↓
4. Information der Projektbeteiligten
   ↓
5. Dokumentation
   ↓
6. Reaktion auf neue Compliance-Entwicklungen
```

| Schritt | Inhalt |
|---|---|
| **1. Identifizierung relevanter Regelwerke** | Welche externen und internen Regelwerke gelten für das Projekt? |
| **2. Ableitung von Compliance-Anforderungen** | Welche konkreten Anforderungen entstehen daraus? |
| **3. Projektbezogene Compliance-Organisation** | Rollen, Verantwortlichkeiten, Strukturen im Projekt |
| **4. Information der Projektbeteiligten** | Aufklärung, Schulung, Bewusstsein |
| **5. Dokumentation** | Nachweisbarkeit der Einhaltung |
| **6. Reaktion auf neue Compliance-Entwicklungen** | Gesetze ändern sich → Anpassung im laufenden Projekt |

*Detailbeschreibungen der einzelnen Schritte werden im PDF nicht textuell ausgeführt – die Folie zeigt nur die sechs Schritte als Schaubild.*
