# MSI - Terminologie - Definition

> [!info] Kurzdefinition
> Eine Terminologie ist eine für die elektronische Verarbeitung nutzbar gemachte Sammlung von Konzepten und Konzept­identifikatoren. Konzepte und Benennungen stammen aus einem bestimmten Vokabular; zur Computerverarbeitbarkeit werden Konzept­identifikatoren („Codes") hinzugefügt.

## Beschreibung

Wörtlich aus Folie 9:

> „Terminologie: Eine für die elektronische Verarbeitung nutzbar gemachte Sammlung von Konzepten und Konzeptidentifikatoren. Die Konzepte und ihre Benennungen stammen aus einem bestimmten Vokabular, zur Computerverarbeitbarkeit werden Konzeptidentifikatoren (‚Codes') hinzugefügt."

Schlüsselwort: **computerverarbeitbar**. Eine Terminologie ist mehr als ein Vokabular – sie hat zugewiesene Codes und ist damit maschinell nutzbar.

## Bestandteile / Aufbau

- Konzepte (gedankliche Einheiten).
- Benennungen (sprachliche Form).
- Konzept­identifikatoren / Codes.
- Quell-Vokabular.

## Abgrenzung

- **Terminologie ≠ Vokabular**: ein Vokabular ist die reine Begriffssammlung; eine Terminologie ist ein computer­verarbeitbares Vokabular mit Codes.
- **Terminologie ≠ Klassifikation**: eine Terminologie kann eine Klassifikation enthalten, ist aber nicht zwingend hierarchisch und vollständig im Sinn der Klassifikations-Definition.
- **Terminologie ≠ Code System**: laut Vorlesung (Folie 10) ist ein Code System „eine Terminologie mit zugehörigem Regelwerk" – die Terminologie ist also Teil eines Code Systems, aber nicht identisch damit.

## Prüfungsrelevanz

**Typische Definitionsfrage:** Was unterscheidet eine Terminologie von einem Vokabular?

**Anwendungsfrage:** Erklären Sie an einem Beispiel (z. B. LOINC), warum eine Terminologie computer­verarbeitbar ist.

**Diskussionsfrage:** Reicht eine Terminologie aus, um semantische Interoperabilität herzustellen, oder braucht es zusätzlich noch Code Systems und Value Sets?
