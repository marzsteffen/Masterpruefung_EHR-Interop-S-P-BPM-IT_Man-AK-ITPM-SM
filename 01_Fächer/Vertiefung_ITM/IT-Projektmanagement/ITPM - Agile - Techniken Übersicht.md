# ITPM - Agile - Techniken Übersicht

> [!info] Kurzdefinition
> Sammelnote für die kleineren agilen Techniken, die in der Vorlesung kurz aufgeführt werden: Task Board, Daily Stand-up, WIP-Limits, Timeboxing, Geschäftswert, osmotische Kommunikation, Earned Value (im agilen Kontext) und Epic. Die größeren Techniken (User Story, Story Points, Planning Poker, DoD, Burn-Down, Persona) sind als eigene Notes verlinkt.

## Beschreibung

Die Vorlesung führt auf den Folien 15–17 eine ganze Reihe agiler Techniken ein. Größere Techniken haben eigene Notes; in dieser Übersichtsnote werden die kürzeren / unkomplexeren gebündelt – jede mit der Definition aus der Vorlesung.

## Bestandteile / Aufbau

| Technik | Definition laut Vorlesung |
|---|---|
| **Task Board** | Visualisierung von Aufgaben, Unterteilung in „offen", „in Bearbeitung" und „erledigt". |
| **Daily-Stand-up-Meetings** | Tägliche Abstimmung des Teams, Dauer ca. 15 min. |
| **Use Cases** | Anwendungsfälle, die Anforderungen aus Kundensicht beschreiben. |
| **Work-in-Progress-Limits (WIP-Limits)** | Begrenzung von parallelen Aufgaben (um Produktivität zu erhalten). |
| **Timeboxing** | Feste Zeitvorgaben. |
| **Geschäftswert** | Möglichst frühzeitige Erzeugung von Kundennutzen. |
| **osmotische Kommunikation** | Gleichen Informationsstand herstellen. |
| **earned Value (agile Variante)** | Fortschritts- und Budgetkontrolle. |
| **Epic** | Zusammenfassen von verwandten Anwendungsfällen. |

## Abgrenzung

- **Task Board** vs. **Kanban Board:** Kanban-Boards arbeiten typischerweise mit WIP-Limits pro Spalte; das in der Vorlesung beschriebene Task Board ist die Basisform.
- **Daily Stand-up** vs. **Daily Scrum:** im Scrum-Kontext heißt das Meeting **Daily Scrum** (siehe [[ITPM - Scrum - Meetings]]).
- **Earned Value** in agilen Methoden ist nicht identisch mit dem klassischen [[ITPM - Earned Value Management]] (PMBOK).
- **Epic** vs. **User Story:** ein Epic ist eine größere Einheit, die in mehrere Stories zerlegt wird.

## Prüfungsrelevanz

**Typische Definitionsfrage:** Definieren Sie kurz Task Board, WIP-Limit, Timeboxing, osmotische Kommunikation und Epic.

**Anwendungsfrage:** Wann setzen Sie WIP-Limits ein, und welchen Effekt erwarten Sie?

**Diskussionsfrage:** „Osmotische Kommunikation" funktioniert in einem Großraumbüro – wie stellt man das in verteilten Teams sicher?

**Mögliche Anschlussfragen:**
- Wie verhält sich „Geschäftswert" zur Backlog-Priorisierung (siehe [[ITPM - Scrum - Backlog Priorisierung]])?
- Welche Beziehung besteht zwischen Task Board und Sprint Backlog in Scrum?
