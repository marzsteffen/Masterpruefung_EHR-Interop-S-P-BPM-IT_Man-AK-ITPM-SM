# BPM - Methode - Ishikawa-Diagramm und 7M

> [!info] Kurzdefinition
> Das **Ishikawa-Diagramm** (auch **Ursache-Wirkungs-Diagramm** oder Fischgräten-Diagramm) ist eine grafische Methode zur strukturierten Ursachenanalyse von Problemen. Die **7M-Methode** liefert sieben Kategorien (Management, Maschine, Methode, Mitwelt, Mensch, Material, Messung), entlang derer mögliche Ursachen systematisch gesucht werden.

## Beschreibung

Folien 124–126. Eingebettet in einen sechsstufigen Problemlösungs­ansatz (Folie 124):

1. Problembeschreibung
2. Sofortmaßnahme
3. Ursachenanalyse
   - a) Brainstorming
   - b) Ursachen-Wirkungs-Diagramm
   - c) 5 × Warum
4. Korrekturmaßnahme / Lösung
5. Lösung umsetzen
6. Wirksamkeitsüberprüfung der Lösung

### Ishikawa-Diagramm (Folie 125)

Das Diagramm hat eine zentrale horizontale Linie (das **Problem / die Wirkung**) und Schräglinien als „Gräten", die die Ursachen-Kategorien darstellen. An jeder Gräte werden konkrete Ursachen eingetragen. Die Folien zeigen die sieben Kategorien als Gräten.

### Die 7M-Kategorien (Folie 126)

| Kategorie | Verbesserungs-Hebel laut Folie |
|---|---|
| **Management** | Verbesserung der Planung etc. |
| **Maschine** | Bestmögliche Nutzung des EDV-Systems etc. |
| **Methode** | Verbesserung der effektiven Ausnützung von Ressourcen etc. |
| **Mitwelt** | Verbesserung der Arbeitsumgebung etc. |
| **Mensch** | Verbesserung der Mitarbeiterausbildung etc. |
| **Material** | Vereinfachung und Kombination von Arbeit und Materialien etc. |
| **Messung** | Verbesserte Messung und Darstellung der Leistung etc. |

Die 7M sind eine Erweiterung der klassischen 4M (Mensch, Maschine, Methode, Material). Die Folien zitieren als externe Quelle: vgl. Wagner/Patzak, Performance Excellence.

## Bestandteile / Aufbau

```
         Management                Mitwelt          Messung
              \                       |               /
               \-----------------+----+--------------/
                                 |
   Maschine ------------+--------|--------+----------- Material
                                 |
                /----------------+----+----------------\
               /                 |                      \
         Methode               Mensch               PROBLEM
```

## Beispiel

Folie 127 zeigt eine Gruppendiskussion zur 5×Warum-Methode mit der Ausgangsfrage „Warum ist die Pflanze auf dem Schreibtisch verwelkt?". Diese Methode ergänzt Ishikawa: erst Kategorisierung (Ishikawa), dann Tieferbohren (5×Warum) – siehe [[BPM - Methode - 5-Warum]].

Klinisch: Problem „postoperative Pneumonie-Rate steigt". Ursachen-Brainstorming entlang der 7M:
- Mensch: unvollständige Atemtherapie-Einweisung
- Methode: keine standardisierte Mobilisierung
- Maschine: defekte Atemtrainer
- Material: …
- Messung: Pneumonie wird zu spät erfasst
- Mitwelt: feuchte Stationsluft
- Management: keine Kennzahl-Verantwortung

## Abgrenzung

- **Ishikawa ≠ FMEA.** Ishikawa = retrospektive Ursachensuche bei tatsächlichem Problem; FMEA = prospektive Risikoanalyse vor dem Eintritt. Vgl. [[BPM - Methode - FMEA]].
- **7M ≠ 4M.** Die klassische 4M-Variante hat nur Mensch/Maschine/Methode/Material. Die 7M-Variante der Vorlesung ergänzt um Management, Mitwelt, Messung.
- **Ishikawa ist Strukturierungs-, kein Analysetool.** Es ordnet Ursachen, beweist sie aber nicht. Tiefenanalyse braucht zusätzliche Methoden (5-Warum, Datenanalyse).

## Prüfungsrelevanz

**Typische Definitionsfrage:** „Was ist ein Ishikawa-Diagramm, und welche sieben Ursachen-Kategorien gibt es?"

**Anwendungsfrage:** „Bauen Sie ein Ishikawa-Diagramm zum Problem ‚Wartezeit in der präoperativen Aufnahme zu lang‘ auf."

**Diskussionsfrage:** Warum ist die Trennung in 7M wichtig, statt einfach zu brainstormen? – Strukturierung verhindert „Tunnelblick"; jede Kategorie zwingt zur Mit­betrachtung systemischer Ursachen (Management, Mitwelt) statt nur menschlicher Fehler.

**Mögliche Anschlussfragen:**
- Welche sechs Schritte umfasst der Problemlösungs­ansatz der Vorlesung?
- Wie ergänzen sich Ishikawa und 5-Warum?
- Worin liegt der Unterschied zur FMEA?
