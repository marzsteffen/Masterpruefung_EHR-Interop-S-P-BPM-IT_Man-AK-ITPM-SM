# ITPM - PRINCE2 - Thema Fortschritt

> [!info] Kurzdefinition
> Das Thema „Fortschritt" richtet Mechanismen ein für einen repräsentativen Soll-Ist-Vergleich, der Abweichungen und damit verbundene Steuerungsmaßnahmen aufzeigt. Operationalisiert die vier Projektsteuerungsmittel **Planen, Delegieren, Überwachen, Steuern**. Beinhaltet Toleranzen, Ausnahmen, Steuerungsmittel und die Unterscheidung Management- vs. technische Phasen.

## Beschreibung

Aus PRINCE-Buch Folien 23–34 (Kapitel 3.5):

**Zweck:** Eingerichtete Mechanismen für einen **Soll-Ist-Vergleich**, der Abweichungen und damit verbundene Steuerungsmaßnahmen aufzeigt.

**Fortschritt** = Vorankommen innerhalb eines vorher definierten Plans. Kann auf **Projekt-, Phasen-, Arbeitspaketebene** überwacht und gesteuert werden. Ziel: **die richtige Information zum richtigen Zeitpunkt**, um die richtige Entscheidung treffen zu können.

## Bestandteile / Aufbau

### Vier Projektsteuerungsmittel

```
Planen → Delegieren → Überwachen → Steuern
```

→ Werden im Prozess IP erstellt und ab dann verwendet.

### Steuerungsmittel als Werkzeug (Folie 25)

Die jeweils höhere Hierarchiestufe kann mit Steuerungsmitteln:

- bereits erzielten **Fortschritt überwachen**
- **Soll-Ist-Abgleiche** aufstellen
- bei Bedarf **steuernd eingreifen**
- **Korrekturmaßnahmen neu planen**
- das Geplante **delegieren**

## Abgrenzung

- **Thema Fortschritt** verbindet sich eng mit **klassischer Projektüberwachung** ([[ITPM - Projektüberwachung - Grundlagen]]) und **Earned Value Management** ([[ITPM - Earned Value Management]]).
