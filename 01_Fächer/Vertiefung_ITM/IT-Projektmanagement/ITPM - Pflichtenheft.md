# ITPM - Pflichtenheft

> [!info] Kurzdefinition
> Das Pflichtenheft (System Requirements Specification) wird vom **Auftragnehmer** erstellt und füllt das Lastenheft mit **Lösungsideen und Aufwänden**. Es ist letztendlich die Quelle für den **Abnahmetest**.

## Beschreibung

Die Vorlesung führt das Pflichtenheft im Requirements-Engineering-Kontext (Folie 73) ein, gemeinsam mit dem Lastenheft.

Zentrale Aussagen:

- Das Pflichtenheft wird durch den **Auftragnehmer** erstellt.
- Es wird mit **Lösungsideen und Aufwänden** gefüllt.
- Lastenheft und Pflichtenheft sollten **dieselbe Inhaltsstruktur** haben (erleichtert den Abgleich).
- Das Pflichtenheft ist letztendlich die Quelle für den **Abnahmetest**.

## Bestandteile / Aufbau

*Konkrete Pflichtinhalte werden im PDF nicht separat aufgeführt* – die Vorlesung verweist darauf, dass die Struktur dieselbe wie beim Lastenheft sein soll (siehe [[ITPM - Lastenheft]]).

## Abgrenzung

| | Lastenheft | Pflichtenheft |
|---|---|---|
| **Wer erstellt?** | Auftraggeber | Auftragnehmer |
| **Sicht** | Anwender / Kunde | Lieferant / Lösung |
| **Frage** | „Was soll geliefert werden?" | „Wie wird es geliefert?" |
| **Inhalt** | Anforderungen, Randbedingungen | Lösungsideen, Aufwände |
| **Funktion** | Ausschreibungsgrundlage | Grundlage für Abnahmetest |
| **Englisch** | User Requirements Specification | System Requirements Specification |

## Prüfungsrelevanz

**Typische Definitionsfrage:** Worin unterscheidet sich Pflichtenheft vom Lastenheft – wer erstellt was, wofür wird es verwendet?

**Anwendungsfrage:** Sie haben das Lastenheft für ein Patientenportal vom Auftraggeber erhalten. Was ist Ihre erste Reaktion als Auftragnehmer?

**Diskussionsfrage:** Wann macht es Sinn, Last- und Pflichtenheft in einem gemeinsamen Dokument zu führen? Welche Risiken birgt die Trennung?

**Mögliche Anschlussfragen:**
- Wie verhält sich das Pflichtenheft zum Abnahmetest im V-Modell?
- Welche Rolle spielt das Pflichtenheft bei vertraglich vereinbarten Festpreisprojekten?
- Wie verhält sich das Pflichtenheft zum Sprint Backlog in agilen Projekten?
