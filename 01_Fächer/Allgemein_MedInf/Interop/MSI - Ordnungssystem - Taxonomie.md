# MSI - Ordnungssystem - Taxonomie

> [!info] Kurzdefinition
> Streng monohierarchisches Ordnungssystem – jeder Begriff hat genau einen Eltern­begriff.

## Beschreibung

Folie 7 listet „Taxonomie" als eines der Ordnungssysteme; eine ausgeschriebene Definition fehlt auf Folie 7. Konkretisiert wird sie auf Folie 17:

> „Streng monohierarchisch: Taxonomie oder Polyhierarchien"

Eine Taxonomie ist also ein Ordnungssystem, in dem jedes Konzept genau einen unmittelbaren Vorgänger in der Hierarchie hat. Im Gegensatz dazu erlaubt eine Polyhierarchie, dass ein Konzept mehrere Eltern hat (Folie 17 zeigt Cimino's Fig. 1: Beispiele a–f für mehrfache Eltern und Konvertierung in strikte Hierarchien).

> Hinweis Quelltreue: Eine umfassendere Definition von Taxonomie liefert das PDF nicht. Im Cimino-Paper (separates PDF) wird das Thema vermutlich vertieft.

## Bestandteile / Aufbau

- Wurzel-Konzept.
- Strenge Vater-Kind-Beziehungen (genau ein Vater pro Konzept).
- Hierarchische Anordnung.

## Beispiel

Aus Folie 4 (historisch): Carl von Linné, *Systema Naturae*, 1758 – klassische biologische Taxonomie als historisches Beispiel der Ordnungs­systeme im Gesundheitswesen.

## Abgrenzung

- **Taxonomie ≠ Polyhierarchie.** Cimino verlangt explizit Polyhierarchy als Desideratum (Folie 14) – eine reine Taxonomie reicht laut Cimino nicht.
- **Taxonomie ≠ Klassifikation.** Klassifikation muss nicht zwingend monohierarchisch sein.
- **Taxonomie ≠ Ontologie.** Ontologie kann beliebige formale Beziehungen abbilden, Taxonomie nur strikte Vater-Kind.

## Prüfungsrelevanz

**Typische Definitionsfrage:** Was unterscheidet Taxonomie von Polyhierarchie?

**Anwendungsfrage:** Geben Sie ein medizinisches Beispiel, das in einer Polyhierarchie sinnvoll, in einer reinen Taxonomie aber problematisch ist (z. B. „Linkes unteres Lungenlappen-Pneumonia" – sowohl als Pneumonie- als auch als anatomisches Konzept einsortierbar).

**Diskussionsfrage:** Warum verlangt Cimino Polyhierarchy als Desideratum?

**Mögliche Anschlussfragen:**
- Welche Probleme entstehen beim Aufbrechen einer Polyhierarchie in eine Taxonomie? (Cimino Fig. 1 d–f auf Folie 17.)
- Wie löst SNOMED CT das Polyhierarchy-Erfordernis?
- Wie verhält sich Taxonomie zur Klassifikation in ICD-10? (ICD-10 ist eine Klassifikation, deren primäre Achse aber wesentlich monohierarchisch organisiert ist.)
