# MSI - Ordnungssystem - Nomenklatur

> [!info] Kurzdefinition
> Sammlung von Fachtermini aus einem bestimmten Fachgebiet mit Richtlinien zur Benennung; ein kontrolliertes Vokabular.

## Beschreibung

Wörtlich aus Folie 7:

> „Nomenklatur: Sammlung von Fachtermini aus einem bestimmten Fachgebiet mit Richtlinien zur Benennung; ein kontrolliertes Vokabular."

Eine Nomenklatur regelt, *wie* in einem Fachgebiet benannt wird. Sie ist die Stufe über dem Glossar in der semantischen Treppe (Folie 8).

## Bestandteile / Aufbau

- Fachtermini eines Gebiets.
- Benennungs-Richtlinien.
- Kontrolliertes Vokabular (Synonyme entweder vermieden oder reglementiert).

## Beispiel

Im PDF dieser Folie nicht explizit ausgeführt. Aus späteren Folien implizit: SNOMED-Bezeichnungen folgen einer Nomenklatur (Fully Specified Name etc.), HGNC ist eine Gen-Nomenklatur (im Paper Lehne erwähnt).

## Abgrenzung

- **Nomenklatur vs. Klassifikation**: Nomenklatur regelt Benennung, Klassifikation ordnet planmäßig in Klassen ein. Eine Nomenklatur muss nicht hierarchisch sein.
- **Nomenklatur vs. Thesaurus**: Thesaurus ergänzt eine Nomenklatur um Definitionen, Synonyme und Hinweise (Folie 7).
- **Nomenklatur ≠ Glossar**: Glossar ist eine einfache Worterklärung, Nomenklatur regelt aktiv die Bezeichnungs-Vergabe.

## Prüfungsrelevanz

**Typische Definitionsfrage:** Was unterscheidet eine Nomenklatur von einem Glossar?

**Anwendungsfrage:** In welcher Domäne arbeiten Sie typischerweise mit einer Nomenklatur, in welcher mit einer Klassifikation?

**Diskussionsfrage:** Warum kann eine Nomenklatur ohne hierarchische Ordnung trotzdem nützlich sein?

**Mögliche Anschlussfragen:**
